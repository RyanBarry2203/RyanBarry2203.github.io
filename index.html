Dying Fire

XAML, WPF

C# Visual Studio

Key Technical Implementations: Architecture: Utilized a robust OOP class hierarchy to manage entities, employing Polymorphism and Inheritance to distinguish between Player, Enemy (BaseDemon), and Interactive Objects. Inventory System: Built a dynamic inventory management system using ObservableCollections and LINQ queries for real-time sorting and filtering of item objects. Data Persistence: Implemented JSON parsing for complex dialogue trees (simulating NPC interactions) and SQL database integration to persist player stats ("Hellflame") and progression. Game Logic: Developed a Finite State Machine (FSM) to handle the transition between the "Investigation Phase" (Point-and-Click) and "Descent Phase" (Turn-Based Combat). Error Handling: Integrated defensive coding practices within the combat calculator to manage exception handling for null references and overflows.
